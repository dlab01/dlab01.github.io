## 1. 목표

학생이 주차별 Google Sheet를 **직접 편집하지 않고**, 웹에서 문제를 보고 답을 제출한다.
제출한 답은 그 주차 시트의 1교시 탭, **본인 행에 자동 기록**된다.
학생은 **다른 학생의 답을 볼 수 없다**. 사이트 전체를 [DESIGN.md](DESIGN.md) 언어로 개편한다.

비목표(v1 범위 밖): 이미지/파일 업로드, Drive 연동, OAuth 로그인, 자동 채점.

## 2. 아키텍처 (확정)

```
[학생 브라우저]                  [GitHub Pages: 정적 프런트]
  answer.html?week=N  ──fetch──▶  (서버 없음)
        │                               │
        │  fetch (GET/POST)             │
        ▼                               ▼
[Google Apps Script Web App]  ──▶  [주차 Google Sheet · 1교시 탭]
        │                          [마스터 시트 · 학생 탭(이름,PIN)]
```

- **프런트**: Vanilla HTML/CSS/JS. 빌드 없음. 푸시 = 배포.
- **백엔드**: Apps Script Web App **1개**, 한 번 배포 후 모든 주차 처리.
- **DB 없음**. 모든 상태는 시트에 있다.

## 3. 데이터 모델 — 라이브 시트 실측 (계약)

주차 시트는 **반드시 네이티브 Google Sheet**다 (업로드 `.xlsx` 금지. §7 참조).

**1교시 탭 구조 (실측 확정):**

```
        A      B        C        D          E            …     (D열부터 질문)
행 1   번호   이름     학년    [질문1]    [질문2]      … [질문N]
행 2   예시)  홍길동   7학년    ← 예시줄. 무시
행 3   박노아          …       답…        답…          …
행 N   …
```

- **헤더(1행): A=`번호`, B=`이름`, C=`학년`, D열~ = 질문.** 질문 개수는 주차마다 다를 수 있다(파서가 동적 처리).
- **2행 = 예시줄(홍길동)** → 항상 스킵.
- **3행~ = 학생.** 이름은 **B열**(매칭 키, A=번호·C=학년은 무시), 답은 **D열부터**.
- 질문 헤더에 셀 내 줄바꿈이 섞일 수 있다 → 파서가 정리.
- 헤더에 "이미지"가 포함된 문항은 v1에서 `type:"image"` + 비활성(교실에서 직접 제출).

**학생 명단:** v1은 프런트 하드코딩 리스트(박노아 / 김세임 / 정재윤 / 한아라). 추후 DB/시트로.

> **v1 식별 정책:** 제출 전용(submit-only) + 이름 선택. PIN은 보류한다 —
> 화면이 다른 학생 답을 절대 불러오지 않으므로(읽기 경로 없음) 프라이버시는 유지된다.
> "내 답 이어쓰기/재방문" 읽기 경로를 추가하는 시점에 PIN(또는 토큰)을 도입한다.

## 4. API 계약 (Apps Script Web App)

베이스 URL은 배포 후 프런트 상수 `WEBAPP_URL`에 저장.

### GET — 질문 조회
```
?action=questions&sheetId=<주차시트ID>
→ 200 { questions: [ { col:4, text:"...", type:"text"|"image", disabled:false } ] }
```
D열(col=4)부터. A·B·C(번호·이름·학년) 제외. 셀 줄바꿈 정리. "이미지" 헤더는 `type:"image", disabled:true`.
`col`은 **절대 열 번호**(D=4, E=5 …)로, 제출 시 그대로 돌려보낸다.

### POST — 답 제출
```
Content-Type: text/plain        ← CORS preflight 회피용 (§7)
body(JSON): { action:"submit", sheetId, name, answers:{ "4":"...", "5":"...", ... } }
→ 200 { ok:true, savedAt:"ISO8601", row:3 }
```
1교시 탭 **B열**에서 `name` 행 탐색(3행~, 없으면 새 행 append) → B에 이름, `col`별 셀에 답 기록.

### (보류) GET — 내 답 조회 (이어쓰기/재방문)
읽기 경로는 v1 범위 밖. 추가 시 PIN/토큰으로 본인 행만 반환(남의 답 절대 미반환).

## 5. 프런트 화면

| 페이지 | 역할 | 변경 |
|--------|------|------|
| `index.html` | 주차 그리드 | 주차 셀이 raw 시트 대신 `answer.html?week=N`로 연결. DESIGN.md 적용. |
| `answer.html` (신규) | 이름+PIN → 질문 폼 → 제출 | 로딩/에러/빈/성공 상태 전부 구현. |
| `student/*` | 알림장 | 기능 동일, DESIGN.md 적용. |

- 주차 잠금 로직(미래 주차·링크 없는 주차)은 기존 [index.html](index.html) 동작 유지.
- `week → sheetId` 매핑은 프런트 한 곳에서 관리(현재 `LINKS`와 동일 위치), 요청에 전달.

### 5.1 answer.html UX (확정 — Typeform형 스텝퍼)

지루(구글폼)·번거로움(시험형)·루즈함(단순 넘기기)을 동시에 피하는 방향.

```
이름 선택 → PIN(1234) → [개요/문항 네비] → 한 문제 한 화면 → 검토·제출
```

- **한 문제 한 화면.** 들쭉날쭉한 문항(1줄 ~ 코드 전체 붙여넣기)을 시원하게 처리, 모바일 친화.
- **상단 진행바 + N/총개수.** 루즈함 제거.
- **문항 네비게이터(항상 노출).** 모든 문항을 번호 그리드로 표시(완료=채워짐 / 빈칸 구분),
  아무 번호나 탭하면 즉시 이동. → "번호 왔다갔다 피로감 0" 요구 반영.
- **전환은 부드럽게.** 페이지 리로드 없이 슬라이드/페이드, 키보드 ←/→ · 스와이프 지원.
- **각 문항 화면은 예쁘게.** DESIGN.md 토큰으로 한 카드에 집중형. (이동 전 "예쁘게 보임" 요구)
- **자동저장(localStorage).** 나갔다 와도 유지 → "왔다갔다 번거로움" 제거.
- **마지막 검토·제출 화면.** 전체 답 한눈에 확인 후 제출(시험형의 장점만 흡수).
- **PIN**: 전부 `1234`(프런트 게이트, 속도방지턱 수준). 학생별 비번 도입 시 백엔드(`명단` 탭)로
  이전하기 쉽게 분리 구현. 읽기 경로 없음 → 프라이버시 유지.

## 6. 필수 상태 처리 (모든 fetch 화면)

- **로딩**: 질문 불러오는 중 / 제출 중 스피너.
- **에러**: 네트워크 실패, `BAD_PIN`, 서버 에러 → 사용자 친화 메시지 + 재시도.
- **빈 상태**: 질문 0개, 아직 제출 안 한 문항.
- **성공**: 제출 완료 + 저장 시각 표시.
- **재제출**: 마감 전 덮어쓰기 허용(myanswers로 기존 답 prefill).

## 7. 알려진 함정 (구현 전 필독)

1. **`.xlsx`는 못 연다.** `SpreadsheetApp.openById()`는 업로드된 엑셀에서 에러.
   주차 시트는 반드시 네이티브 Google Sheet. (실측: 기존 파일은 `.xlsx`였음 → 전환 확정)
2. **CORS preflight.** Apps Script Web App에 `application/json`으로 POST하면 preflight가
   막힌다. POST는 `Content-Type: text/plain`으로 보내고 `e.postData.contents`를 JSON 파싱.
3. **이름 매칭.** 주차 시트 A열 이름 ≠ 마스터 명단 이름이면 매칭 실패. 표기 통일 필수.
4. **Apps Script 쿼터/지연.** 동시 제출 많으면 느릴 수 있음. 제출 중 버튼 비활성으로 중복 방지.
5. **배포 버전.** Apps Script는 "배포 관리"에서 새 버전을 내야 코드 변경이 반영됨(저장만으론 X).

## 8. 구현 순서 (검증 게이트 포함)

1. 마스터 `학생` 탭(이름·PIN) 생성 + 1교시 1행 계약 확정 → 검증: 샘플 시트로 파싱 확인
2. Apps Script `doGet`(questions/myanswers) → 검증: URL 직접 호출로 JSON 확인
3. Apps Script `doPost`(submit) → 검증: 시트에 본인 행이 실제로 채워지는지 육안 확인
4. `answer.html` 신규(모든 상태 포함) → 검증: 정상/오류 PIN/네트워크 실패 시연
5. `index.html` 개편(answer 연결 + DESIGN.md) → 검증: 잠금 로직 유지 + 모바일
6. `student/*` 개편(DESIGN.md) → 검증: 기능 동일
7. 공유 CSS 토큰 파일(DESIGN.md custom properties) → 검증: 3페이지 일관성
